html

语法

  • 嵌套元素应当缩进一个制表符
  • 缩进采用4个空格制表符（有可能表现不一致）
  • 所有属性定义应当使用双引号，不要使用单页号
  • 不要在自闭合标签尾部添加反斜线
  • 不要省略可选的结束标签
  • 统一使用utf8字符编码
  • 所有boolean属性只需要写属性名称，无须赋值
  • 可省略属性尽量省略

<!DOCTYPE html>

<html>

<head>

<meta charset="utf-8">

<title>page title</title>

<link rel="stylesheet" src="path">

<script src="path"></script>

</head>

<body>

...

</body>

<html>

文档声明

为每个 HTML 页面的第一行添加标准模式（standard mode）的声明，这样能够确保在每个浏览器中拥有一致的展现。

<!DOCTYPE html>

代码注释

一个好的书写习惯可以提高代码的可读性，那么添加注释必不可少

  • html采用 <!-- -->
  • css采用 /* ... */
  • js采用 //...、/* ... */

<--这是一个标题-->

<h1>title</h1>

<style>

/*这里是一段css代码*/

</style>

<script>

//这里是一段JS代码

</style>

属性顺序

HTML 属性应当按照以下给出的顺序依次排列，确保代码的易读性。

  • id，name
  • class
  • width，height
  • data-*
  • src,href,type,for
  • title, alt

id，name为独立私有的属性，而class具有复用性，所以要比id，name低

<a class="class" href="path" title="title">

<img src="path" alt="title">

IE 兼容模式

IE 支持通过特定的 标签来确定绘制当前页面所应该采用的 IE 版本。除非有强烈的特殊需求，否则最好是设置为 edge mode，从而通知 IE 采用其所支持的最新的模式。

<meta http-equiv="X-UA-Compatible" content="IE=Edge">

SEO相关

  • 头部注意书写关键字与描述性息
  • img标签注意属性alt属性
  • 注意标题标签的使用次数
  • a标签建议填写title属性（有溢出省略时用之最好）

<meta name="keywords" content="key">

<meta name="description" content="description">

<h2>页面中只出现3-4次的标题(描述的是每一屏/模块的标题)</h2>

<h3> - <h6>次数没有固定要求

css

语法

  • 嵌套元素应当缩进一个制表符
  • 为了代码的易读性，在每个声明块的左花括号前添加一个空格。
  • 所有声明语句都应当以分号结尾。最后一条声明语句后面的分号是可选的，但是，如果省略这个分号，你的代码可能更易出错。
  • 对于以逗号分隔的属性值，每个逗号后面都应该插入一个空格
  • 不要在 rgb()、rgba()、hsl()、hsla() 或 rect() 值的内部的逗号后面插入空格。这样利于从多个属性值（既加逗号也加空格）中区分多个颜色值（只加逗号，不加空格）。
  • 对于属性值或颜色参数，省略小于 1 的小数前面的 0
  • 尽量使用简写形式的十六进制值
  • 避免为 0 值指定单位
  • 为了获得更准确的错误报告，每条声明都应该独占一行
  • 每条声明语句的 : 后应该插入一个空格。

/* Bad CSS */

.selector, .selector-secondary, .selector[type=text] {

padding:15px;

margin:0px 0px 15px;

background-color:rgba(0, 0, 0, 0.5);

}

 

 

/* good CSS */

.selector,

.selector-secondary,

.selector[type=text] {

padding: 15px;

margin: 0 0 15px;

background-color: rgba(0,0,0,.5);

}

属性声明顺序

相关的属性声明应当归为一组，并按照下面的顺序排列：

  • Positioning
  • Box model
  • Typographic
  • Visual

css3提供的特效个人觉得应该属于Visual里面，所有大家在它后面写就好了

.selector {

/* Positioning */

position: absolute;

top: 0;

right: 0;

bottom: 0;

left: 0;

z-index: 1000;

 

/* Box model */

display: box;

float: left;

width: 100px;

height: 100px;

padding: 0;

border: 1px solid #ddd;

margin: 0;

 

/* Typographic */

font: normal 14px/1.5 "Helvetica Neue", sans-serif;

color: #666;

text-align: center;

 

/* Visual */

background: color img;

opacity: .5;

}

注释

代码是由人编写并维护的。请确保你的代码能够自描述、注释良好并且易于他人理解。好的代码注释能够传达上下文关系和代码目的。不要简单地重申组件或 class 名称

 

对于较长的注释，务必书写完整的句子；对于一般性注解，可以书写简洁的短语。

 

如果是多人合作的项目，在写注释时一定要加上描述、作者、时间

/*

description：以下到下一个同样的注释前，是文章页的所有样式

ather：张三

update：2015-10-9

*/

/* 文章主体内容样式 */

.selector {

position: absolute;

top: 0;

right: 0;

bottom: 0;

left: 0;

z-index: 1000;

}

class 命名

  • 可以根据当前区域添加前缀命名
  • 可以参考设计图的模块进行命名
  • 不要使用拼音，尽量用英文命名
  • class 名称应当尽可能短，并且意义明确
  • 使用破折号-连接

/*网站头部*/

.site-head {

...

}

 

.site-head .title {

...

}

js

语法

  • 缩进一个制表符
  • 变量声明都写在头部
  • 变量名过多时，换行书写
  • 函数左大括号之前空一个空格，右大括号另起一行
  • 函数可以使用函数表达式与变量声明写在一起
  • 使用驼峰命名
  • 使用单引号
  • 运算符2边添加空格

var name = '张三',

age = 28,

sex = '女',

showHello = function() {

alert('hello word');

};

注释

  • 方法、接口采用JSDOC注释格式
  • 对于复杂业务逻辑写详细注释
  • 没必要每行都写注释

/*

@author zhangsan

@description 一个选项卡切换特效

@since version 0.1

@param {Arr} titles 选项卡所有标题集合

@param {Arr} contents 选项卡所有内容集合

*/

function tabChange(titles, contents) {

...

}

变量命名

  • 变量命名应该语义化
  • 为方便了解变量类型，应加上变量类型后缀（arrary = Arr string = str object = Obj function = fun number = Num boolean = Bool （jquery = $为前缀））

function tabChange(titles, contents) {

var titleObj = document.getElementById('id');

$dom = $("#id");
